# CRM系统详细设计文档

# 1 系统概述

本CRM系统旨在实现客户全生命周期管理、销售流程规范化、数据统计可视化，适配5000用户规模，支持本地Docker部署，采用分层架构设计，包含客户管理、销售管理、数据统计分析、接口对接、辅助功能、系统管理6大核心模块，各模块协同工作，支撑企业客户管理与销售业绩提升，同时预留灵活扩展接口，适配未来业务迭代需求。

# 2 核心模块详细设计

## 2.1 客户管理服务详细设计

### 2.1.1 模块功能

负责客户信息的全生命周期管理，包括客户新增、编辑、查询、删除、归档，客户标签管理、客户跟进记录管理，以及公海客户的分配与认领，适配现有角色权限体系（如管理员可查看所有客户、销售仅查看个人负责客户），支撑5000用户规模的客户数据存储与并发访问，适配本地Docker部署的数据存储要求，确保客户数据的安全性与一致性。

### 2.1.2 核心流程

#### （1）客户新增流程

1. 拥有“客户新增”权限的用户（如销售、管理员），通过前端界面填写客户基础信息（名称、联系方式、行业、地址等），可选择添加客户标签；

2. 前端校验信息合法性（如联系方式格式、客户名称非空），校验通过后提交请求至服务层；

3. 服务层接收请求，再次校验权限与信息完整性，关联现有登录用户ID作为客户归属人，生成客户唯一ID；

4. 数据层将客户信息写入数据库，同步记录操作日志，返回新增成功提示及客户ID，前端更新页面展示。

#### （2）客户跟进流程

1. 客户归属人或拥有“跟进权限”的用户，针对指定客户发起跟进操作，填写跟进内容、跟进类型（电话、微信、面谈等）、下次跟进时间；

2. 服务层校验用户权限（仅归属人及管理员可跟进）、客户状态（非归档客户可跟进），校验通过后，新增跟进记录，关联客户ID与操作人ID；

3. 同步更新客户表的“最后跟进时间”字段，若填写下次跟进时间，触发辅助功能服务的提醒机制；

4. 返回跟进成功提示，前端更新客户跟进记录列表，支持按时间排序查看。

#### （3）公海客户认领流程

1. 销售用户查看公海客户列表（无归属人的客户），选择目标客户发起认领请求；

2. 服务层校验用户权限（仅销售可认领）、客户状态（公海状态），确认客户未被其他用户认领；

3. 更新客户表的“归属人ID”字段，将客户状态从“公海”改为“待跟进”，同步记录认领日志；

4. 返回认领成功提示，前端更新公海客户列表与个人客户列表，同步推送提醒至认领人。

### 2.1.5 异常处理

- 权限异常：无对应操作权限（如普通销售查看其他销售客户），返回权限不足提示，拦截操作，适配现有权限校验逻辑；

- 信息异常：客户名称为空、联系方式格式错误或重复，返回明确错误提示，阻止新增/修改，引导用户修正；

- 状态异常：认领已被认领的公海客户、跟进已归档的客户，返回状态错误提示，明确当前客户状态；

- 数据异常：客户ID不存在、标签ID无效，返回数据不存在提示，同时记录异常日志，便于排查。

## 2.2 销售管理服务详细设计

### 2.2.1 模块功能

负责销售全流程管理，包括商机管理、合同管理、销售业绩统计等功能，适配现有角色权限体系（如销售经理可查看团队业绩、普通销售仅查看个人业绩），衔接客户管理服务（关联客户生成商机/合同），规范销售流程，提升成单效率，支撑5000用户规模的销售管理需求，适配本地Docker部署的业务数据存储要求。

### 2.2.2 核心流程

#### （1）商机管理流程

1. 拥有“商机新增”权限的用户（如销售），关联现有客户，发起商机新增请求，填写商机名称、预估金额、所处阶段等信息；

2. 服务层校验客户合法性、操作人权限，校验通过后，新增商机，关联客户ID与操作人ID，记录商机创建时间；

3. 商机推进时，操作人（商机归属人）更新商机阶段（如从“初步沟通”至“方案报价”），填写推进备注，系统记录阶段变更日志；

4. 商机成单后，关联生成合同；商机失败/终止时，填写失败原因，归档商机，同步更新客户状态。

#### （2）合同管理流程

1. 商机成单后，销售（拥有“合同新增”权限）关联商机与客户，发起合同新增请求，填写合同编号、金额、签订日期、有效期等信息，上传合同附件（对接辅助功能服务）；

2. 服务层校验合同信息合法性（如合同编号唯一性、金额非负）、操作人权限，校验通过后，新增合同，初始状态设为“待审核”；

3. 拥有“合同审核”权限的用户（如销售经理），查看合同详情，进行审核（通过/驳回），填写审核意见；

4. 审核通过后，合同状态更新为“已生效”，同步生成销售业绩；审核驳回时，返回驳回原因，引导销售修改后重新提交；

5. 合同生效后，可进行合同变更、终止操作（需对应权限），所有操作均记录日志，便于追溯。

#### （3）销售业绩统计流程

1. 系统每日凌晨自动统计前一日销售业绩，关联已生效合同，按销售用户、部门分组计算业绩金额、成单数量；

2. 拥有“业绩查看”权限的用户，可按时间范围（日/周/月/年）、部门、用户查询业绩数据，生成基础业绩报表；

3. 销售经理可查看团队所有成员业绩，普通销售仅查看个人业绩，适配现有角色权限分级管控；

4. 业绩数据支持导出（对接辅助功能服务），便于线下统计与分析。

### 2.2.5 异常处理

- 权限异常：无商机/合同操作权限、普通销售查看他人业绩、无审核权限进行审核操作，返回权限不足提示，拦截操作，适配现有权限体系；

- 信息异常：合同编号重复、金额为负、签订日期晚于生效日期，返回明确错误提示，阻止新增/修改，引导用户修正；

- 状态异常：审核已生效/已驳回的合同、更新已归档/失败的商机、终止未生效的合同，返回状态错误提示，明确当前业务状态；

- 数据异常：关联的客户ID、商机ID不存在，合同附件上传失败，返回数据不存在或操作失败提示，记录异常日志，便于排查；

- 并发异常：多个用户同时认领同一商机、同时审核同一合同，触发并发控制，返回“操作冲突，请稍后重试”提示，避免数据错乱，通过分布式锁保障数据一致性；

- 附件异常：合同附件格式不支持、附件大小超出限制（默认不超过100M），返回附件异常提示，明确支持的格式（PDF、Word、Excel）及大小限制，引导用户重新上传；

- 业绩异常：合同生效后未自动生成业绩、业绩金额与合同金额不一致，自动触发异常校验机制，同步修正业绩数据，同时发送提醒至系统管理员，记录异常详情供排查；

- 时间异常：商机创建时间晚于跟进时间、合同生效时间晚于终止时间，返回时间逻辑错误提示，引导用户修正时间参数，确保业务流程时间逻辑合理。

# 3 数据统计分析服务详细设计

## 3.1 模块功能

负责CRM系统全量数据的统计与分析，衔接客户管理、销售管理模块，实现客户数据、销售数据、业绩数据的可视化呈现与多维度分析，支撑企业经营决策。核心功能包括客户统计、销售统计、业绩分析、数据报表生成与导出，适配现有角色权限体系（管理员查看全量数据、销售经理查看团队数据、普通销售查看个人数据），支持按时间、部门、用户等多维度筛选统计，适配5000用户规模的数据统计需求，确保统计结果实时、准确，适配本地Docker部署的数据处理规范。

### 3.1.1 核心流程

#### （1）数据统计流程

1. 系统按定时任务（每日凌晨）自动同步客户管理、销售管理模块的全量数据，包括客户新增/流失数据、商机推进数据、合同生效数据、业绩数据等，进行数据清洗与校验，剔除异常数据；

2. 按预设统计维度（时间维度：日/周/月/年；组织维度：全公司/部门/个人；业务维度：客户/商机/合同/业绩）进行数据聚合计算，生成基础统计数据集；

3. 统计数据实时同步至前端可视化界面，支持用户按自定义条件筛选、排序，适配不同角色的数据查看权限；

4. 定时任务执行异常时，触发提醒机制，发送异常通知至系统管理员，记录异常日志，便于排查修复，确保统计数据的连续性。

#### （2）数据报表生成与导出流程

1. 用户（拥有对应权限）在前端选择报表类型（客户报表、销售报表、业绩报表）、统计时间范围、筛选条件，发起报表生成请求；

2. 服务层校验用户权限、筛选条件合法性，调用数据统计接口获取对应数据集，按预设报表模板（表格/图表）组装报表内容；

3. 报表生成完成后，前端展示报表详情，支持在线查看、打印；用户发起导出请求时，对接辅助功能服务，生成Excel/PDF格式报表文件，返回文件下载地址；

4. 记录报表生成、导出操作日志，关联操作人、操作时间、报表参数，便于追溯与审计。

### 3.1.2 核心接口设计

|接口名称|请求方式|请求参数|响应参数|接口说明|
|---|---|---|---|---|
|客户数据统计|GET|timeRange、deptId（可选）、userId（可选）、token|success、message、data（新增/流失/活跃客户数据）|多维度统计客户数据，按权限返回对应范围数据|
|销售数据统计|GET|timeRange、deptId（可选）、userId（可选）、token|success、message、data（商机/合同相关统计数据）|统计商机推进、合同签订/生效数据，支持多维度筛选|
|业绩数据统计|GET|timeRange、deptId（可选）、userId（可选）、token|success、message、data（业绩金额/成单数量统计）|按部门/个人统计业绩，支持同比、环比分析|
|报表生成|POST|reportType、timeRange、filterData、token|success、message、reportId、reportContent|生成指定类型报表，返回报表详情与唯一ID|
|报表导出|GET|reportId、exportType、token|success、message、fileUrl|导出指定报表，支持Excel、PDF格式，返回下载地址|
